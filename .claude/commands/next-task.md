请完成以下流程：

1. 读 `docs/M1.md` §4 任务卡总表，找出**第一张状态为 ⬜ 或 🟡 的任务卡**（从 T0' 开始往后扫）
2. **先调用 `/check-prerequisites TX`** 确认前置知识齐全（缺的自动补 knowledge 卡）
3. 如果该任务卡在 `docs/tasks/` 还没有详细文件，按 `_TEMPLATE.md` 生成完整 7 字段内容并写入 `docs/tasks/M1-TX-*.md`：
   - 前置（依赖哪些 T + 必读的 knowledge 卡）
   - 产出文件
   - 算法核心（代码骨架，留 TODO 让作者填）
   - L0 测试清单（表格）
   - DoD（验收标准）
   - 坑（按概率排序，引用 lessons/ 中相关条目）
   - 估时
4. 输出给用户：
   - 任务卡标题 + 一句话目标
   - **前置阅读清单**（来自 step 2）
   - 算法核心代码骨架
   - L0 测试用例数
   - 启动 checklist（从 `docs/M1.md` §10 抄）
5. **不要**自己写 `inferlite/` 下的实现代码（见 `CLAUDE.md` 反模式 #1）
6. 等用户写完代码 + 跑测试后再继续

如果所有 M1 任务卡都已 ✅，提示用户进入 M2 并读 `docs/PLAN.md`。
