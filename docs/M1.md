## 4. Protocol 签名（M1 只钉最小契约）

M1 只稳定 `LLMModel`、`Sampler`、`EngineCore.step()` 三个入口。`KVCache / Scheduler / ModelExecutor / Drafter` 可以先留占位或极薄实现，**不要承诺签名稳定**；等 M2/M3/M7 有真实需求后再收紧。

```python
# inferlite/model/protocol.py
class LLMModel(Protocol):
    config: ModelConfig
    def forward(
        self,
        input_ids: Optional[Tensor] = None,
        inputs_embeds: Optional[Tensor] = None,   # M13 VLM 预留
        positions: Tensor = ...,
        kv_cache: "KVCache" = ...,
    ) -> Tensor: ...                              # 返回 logits [B, T, V]
    @classmethod
    def load_from_hf(cls, hf_path: str) -> "LLMModel": ...

# inferlite/cache/protocol.py（M2 再稳定签名，M1 仅占位）
class KVCache(Protocol):
    def append(self, layer_id: int, k: Tensor, v: Tensor, slot: Any) -> None: ...
    def get(self, layer_id: int, slot: Any) -> tuple[Tensor, Tensor]: ...
    def free(self, slot: Any) -> None: ...

# inferlite/scheduler/protocol.py（M3 再稳定签名，M1 仅占位）
class Scheduler(Protocol):
    def schedule(self, requests: list["Request"]) -> "Batch": ...
    def update(self, output: "ModelOutput") -> list["Request"]: ...

# inferlite/sampler/protocol.py
class Sampler(Protocol):
    def __call__(self, logits: Tensor, params: "SamplingParams") -> Tensor: ...

# inferlite/executor/protocol.py（M1b 最小实现，M2/M3 允许调整）
class ModelExecutor(Protocol):
    def execute_model(self, batch: "Batch") -> "ModelOutput": ...

# inferlite/engine/core.py
class EngineCore:
    def step(self) -> list["Request"]:
        batch  = self.scheduler.schedule(self.requests)        # phase A
        output = self.executor.execute_model(batch)            # phase B
        return self.scheduler.update(output)                   # phase C

# inferlite/scheduler/protocol.py（Drafter 留位，M7 才实现并稳定）
class Drafter(Protocol):
    def propose(self, request: "Request") -> list[int]: ...
```

**纪律**：M1b 必须通过 `LLMModel` 进入模型、通过 `Sampler` 选 token、通过 `EngineCore.step()` 串流程；其余协议只做“有地方放”的占位，不为好看而提前设计大接口。

## 6. 易踩坑预警（按概率排序）

| # | 坑 | 原因 | 防御措施 |
| --- | --- | --- | --- |
| 1 | **logits 偏差大但形状对** | RoPE 实现选错（GPT-J 风格 vs LLaMA 风格 vs Qwen 内部 fused 风格） | 严格对照 transformers `Qwen3RotaryEmbedding`，**不要**自己脑补复数版 |
| 2 | **QK-norm 漏写** | Qwen3 在 Q/K 投影后多了一次 RMSNorm（区别于 Qwen2） | `attention.py` 加 `q_norm` / `k_norm`；测试 4 专门盯这个 |
| 3 | **GQA repeat_kv 顺序错** | KV 头复制要 `repeat_interleave(dim=1)` 不是 `repeat` | 用 transformers 的 `repeat_kv` 直接对照 |
| 4 | **tie_word_embeddings 漏接** | Qwen3-0.6B 的 lm_head 与 embed_tokens 共享权重 | `weight_map.py` 显式处理；测试 8 验证 |
| 5 | **MPS dtype 报错** | MPS 在 fp16 某些算子有坑 | M1 全程 fp32（M5 benchmark 才上 fp16/bf16） |
| 6 | **causal mask 没传** | SDPA 必须 `is_causal=True` | 单测在 attention 测试里盖一个 prompt-len > 1 的 case |
| 7 | **HF 权重路径硬编码** | 不同人本地 HF cache 位置不同 | `load_from_hf("Qwen/Qwen3-0.6B")` 走 `huggingface_hub.snapshot_download` |
| 8 | **首次 forward 慢** | MPS JIT warm-up | benchmark 第一次扔掉，从第二次开始计时（M5 才 benchmark，M1 不管） |
| 9 | **国内连不上 huggingface.co** | 域名被墙；`HF_ENDPOINT` 镜像在 `huggingface_hub>=1.x` 已被官方域名硬校验失效 | 默认走 ModelScope（`make preflight` / `inferlite/model/loader.py` 的 source 选项），权重一致；详见 [SETUP §5.4](./SETUP.md) |

## 9. 概念速查（M1 期间常翻）

> 配合 `scripts/preflight.py` 看，preflight 的 5 行代码就是"transformers 推理最小闭环"。

### 9.1 五个核心对象

| 对象 | 作用 | M1 是否要自己写 |
| --- | --- | --- |
| `AutoTokenizer.from_pretrained(id)` | 字符串 → token id 列表（BPE 分词） | 否，永远复用 transformers |
| `AutoModelForCausalLM.from_pretrained(id, dtype, device_map)` | 下载权重 + 实例化网络 | **是**，inferlite 版叫 `LLMModel.load_from_hf("Qwen/Qwen3-0.6B")` |
| `model.eval()` | 切推理模式（关 dropout / batchnorm 更新） | 是，`__init__` 末尾自动调 |
| `tokenizer.decode(ids, skip_special_tokens=True)` | token id → 字符串 | 否 |
| `model.generate(...)` | 自动循环 forward 直到 max_new_tokens 或 EOS | **要手撕**：cli.py 里自己写循环 + 采样 |

`generate(...)` 的本质（M1 你 cli 要重写的）：

```python
while not done:
    out  = model(input_ids)                 # forward 一次
    next = sampler(out.logits[:, -1, :])    # 取最后位置 logits → 采样
    input_ids = torch.cat([input_ids, next], dim=1)
```

### 9.2 关键超参

**generate 超参**：

| 超参 | 含义 |
| --- | --- |
| `max_new_tokens` | 新生成 token 上限（不含 prompt） |
| `do_sample=False` | 贪心解码（M1 唯一支持的策略） |
| `do_sample=True` + `temperature/top_p/top_k` | 随机采样（M5 才做） |
| `num_beams` | beam search（inferlite **不做**） |
| `pad_token_id` | padding 用，单 prompt 单 batch 不需要 |

**from_pretrained 超参**：

| 超参 | 含义 | M1 用值 |
| --- | --- | --- |
| `<owner>/<name>` repo id | HF Hub 仓库 | `"Qwen/Qwen3-0.6B"` |
| `dtype` | 权重精度 | `torch.float32`（MPS 全程 fp32，§6 第 5 坑） |
| `device_map` | 权重搬到哪 | `"mps"` / `"cuda"` / `"cpu"`（自动选见 preflight.py `pick_device`） |

> transformers 5.x 用 `dtype=`；4.x 老教程是 `torch_dtype=`，二者等价。

**张量转换两件事**：

```python
inputs = tokenizer("你好", return_tensors="pt").to(device)
#                                  ^^^                ^^^
#                                  返回 PyTorch tensor   搬到模型所在设备
```

### 9.3 推理优化常用上下文

```python
with torch.no_grad():       # 不算梯度，省 ~50% 显存 + 加速 5–30%
    out = model(...)

# 等价（更彻底）：
with torch.inference_mode():
    out = model(...)
```

M1 推理路径**必须**包一层，否则 MPS 上显存翻倍。

### 9.4 形状速查（背下来）

| 张量 | 形状 | 含义 |
| --- | --- | --- |
| `input_ids` | `[B, T]` | int64 token ID |
| `inputs_embeds` (M13) | `[B, T, H]` | 跳过 embed 直接喂浮点 |
| 隐藏态 | `[B, T, H]` | 每层 attention/MLP 之间 |
| logits | `[B, T, V]` | 每个位置在词表上的"分数" |
| 下一 token | `[B, 1]` | argmax 出来的 ID |

**字母含义**：

- **B** = batch size（M1 = 1；M3 引入连续 batching）
- **T** = sequence length（token 数）
- **H** = hidden_size（Qwen3-0.6B = 1024）
- **V** = vocab_size（Qwen3 = 151,936）
- **N** = layers（Qwen3-0.6B = 28）

**Qwen3-0.6B 全部超参**（写 `ModelConfig` 时抄这些）：

| 字段 | 值 | 来源 |
| --- | --- | --- |
| hidden_size (H) | 1024 | config.json |
| num_hidden_layers (N) | 28 | |
| num_attention_heads | 16 | |
| num_key_value_heads | 8 | GQA 比例 16:8 |
| head_dim | 128 | hidden_size / num_attention_heads |
| intermediate_size | 3072 | MLP 中间维度 |
| vocab_size (V) | 151936 | tokenizer 表大小 |
| max_position_embeddings | 40960 | 原生上下文（YaRN 后到 1M） |
| rope_theta | 1000000.0 | RoPE 基频 |
| rms_norm_eps | 1e-6 | RMSNorm ε |
| tie_word_embeddings | true | lm_head 与 embed_tokens 共享权重 |

写 `ModelConfig` 时把这 11 个字段塞进去就够了，其他 `transformers.Qwen3Config` 里的字段 M1 都不需要。

### 9.5 一句话贯穿前向

```
[B, T] int → embedding → [B, T, H] →
  ┌────────────── 28 × Transformer Block ──────────────┐
  │  RMSNorm → Attention(GQA + RoPE + QK-norm) + 残差  │
  │  RMSNorm → SwiGLU MLP                     + 残差  │
  └────────────────────────────────────────────────────┘
→ Final RMSNorm → lm_head([V, H] = embed_tokens.T) → [B, T, V]
```

> M1 你要从 RMSNorm 开始一个一个手敲，每敲一个加一个 L0 测试 vs transformers 同名实现 allclose。
